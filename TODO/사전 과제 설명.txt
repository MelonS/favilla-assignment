사전 과제 설명

1. 과제 목적 : 설계 능력을 확인하기 위한 간단한 테스트

2. 제출 방법 : 완성된 코드와 코드를 설명하는 README파일을 하나의 압축 파일로 제출

3. 과제 배경
- MTM 프로젝트에서, 인게임 유닛을 설계하는 단계에서 작업중임을 가정합니다.
- 제작해야하는 유닛은 챔피언, 미니언, 타워입니다.
- 단순 설계 작성 능력 확인을 위함이므로 컴파일 되지 않아도 괜찮습니다.
- 60~90분정도의 소요시간을 예상하며 작성하였으며, AI를 활용하셔도 무방합니다.
- 코드 및 README 문서 작성에 있어 텍스트의 '양'은 중요하지 않습니다.
- 구조 설계 이외의 구체적인 구현은 생략하셔도 괜찮습니다. 
 > 예시
   public int GetProperAttackDamageBy(int level) => 0; // 레벨에 맞는 적절한 공격력 반환

4. 과제 (1) - 필수
- 챔피언, 미니언, 타워 class를 작성해주세요. 요구 사항은 다음과 같습니다.
 > 챔피언 : 죽으면 일정 시간 후 부활
 > 미니언 : 죽으면 소멸, 생성시 BattleManager.CurrentGameSeconds에 비례하여 공격력 세팅
 > 타워 : 죽으면 영구 파괴

5. 과제 (2) - 선택, 추가 요소
- 각 유닛 class -> HPBar로의 의존성이 없는 채로, HPBar가 임의의 대상의 최신 hp 상태를 반영할 수 있는 구조를 설계해주세요.
- 첨부된 HPBar.cs의 HPBar는 임의의 유닛의 HP를 노출시킬 UI Object입니다.
- HP에 대한 '보여주는' 객체와 '작동하는' 코어 객체간의 관계와 책임을 어떻게 분리하고 유지할지 자유롭게 설계해주세요.
